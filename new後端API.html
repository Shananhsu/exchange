API

送出注單

  • Path
item value note
Method POST
URI /order
  • Header
header value note
playertoken 玩家的token, 目前開發時暫時先以testtoken字樣做測試
  • Request
param type desc note
symbol string 商品名
longshort int 看漲看跌 1:看漲 -1:看跌
rangeseq int 區間百分比選項
betamount float64 押注金額
auto bool 自動續單
autotype int 自動續單類型 -1:輸續單, 1:贏續單, 2:輸贏皆續單
autodir int 同(反)向續單 1或0:同向續單, -1:反向續單 









{
    "symbol":"EURUSD",    
    "longshort":1, //看漲或看跌
    "rangeseq":1,
    "betamount": 100,//押注金額
    "auto":true,//設定自動續單
    "autotype":1,    //自動續單的類型
    "autodir":-1 //反向續單
}
  • Response
    • 回應只會先有訂單ID, 因為訂單不一定會立刻成交, 必須要等到有下一個報價才能成交
col type desc note
orderid string 訂單ID 











{
    "data":{
        "orderid":"xyzabcdefghi12345",    
    },
    "status":{
        "code":0,
        "msg":"success",
        "dt":1682770433,
        "trace":yuiuyrertyughj
    }
}

取得已結算訂單

  • Path
item value note
Method GET
URI /order/settled
  • Header
header value note
playertoken 玩家的token, 目前開發時暫時先以testtoken字樣做測試
  • Request
    • 參數的使用方式, 這邊分成三種使用方式
    • 第一, 只給symbol, 其餘4個參數都給空值, server端回傳最近七天的已結算歷史訂單
    • 第二, 給symbol, idx 與takecount三個參數, server端會回傳第idx筆資料開始的資料, 取takecount筆資料, 例如從第5筆資料開始取,取10筆資料出來, 那idx給5, takecount給10
    • 第三, 給symbol, start, end三個參數, server端會回傳從start開始到end結束的資料(start跟end的區間不得大於7天)
param type desc note
symbol string 商品代碼
idx int 資料索引 從1開始, 指的是第幾筆資料開始
takecount int 取得資料筆數
start string 開始時間 使用RFC3339格式, 時間請轉為 GMT+0 時區傳送給後端
end string 結束時間 使用RFC3339格式, 時間請轉為 GMT+0 時區傳送給後端
  • Response
col type desc note
orders array 訂單資料陣列

Web Socket

  • Path
item value note
Protocol Websocket
Method GET
URI /symbol/quote/:playertoken
  • 說明
    • Web Socket可以達成由server端主動通知或傳送即時訊息給client端
    • 目前server端會傳送即時的商品報價資料, 還有新的K線資料, 注單成交通知, 注單結算進度…等資料

報價訊息

cmd : Quote

方向 : client <— server

  • 說明:
    • 此訊息是商品的最新報價資料, 最新的成交價格是 last 欄位, ts 是成交的時間戳記(unix timestamp), 可以在網路上找到轉換的工具或網頁(ex. 轉換網頁)














{
    "cmd": "Quote",
    "msgtime": "2023-05-03T16:13:41+08:00",
    "Data": {
        "symbol": "EURUSD", //商品代碼
        "last": 1.10392,  //最新的成交價
        "ts": 1683101123,  //成交時間 timestamp(unix)
        "qtime": "2023-05-03T16:05:23+08:00", //成交時間
        "change": -0.01115, //價格漲跌
        "changepercent": 0.99, //漲跌幅%, 這個數字代表 0.99%
        "refclose": 1.11507, //前一個交易日收盤價
    }
}

K線訊息

cmd : KLine

方向 : client <— server

  • 說明:
    • 此訊息在大部分的時間點只會出現一次, 通常是後端收到IB的新K線資料時, 會由server端傳送給client端作參考, 但需要注意的是, 此K線資料並不一定會非常即時傳送給client端, 例如, 15分K線, 並不一定會即時地在00, 15, 30或是 45分的時候準時發送給client端, 因為IB不見得會這麼即時送出, 因此使用上需要注意
















{
    "cmd": "KLine",
    "msgtime": "2023-05-03T16:13:41+08:00",
    "Data": {
        "symbol": "EURUSD", //商品代碼
        "period": 15, //K線間隔
        "unit": 2, //1:秒 2:分 3:時 4:日 5:周 6:月
        "open": 1.1025, //開
        "high": 1.1026, //高
        "low": 1.10215, //低
        "close": 1.10215, //收
        "ts": 1683089100, //K線時間timestamp(unix)
        "ktime": "2023-05-03T12:45:00+08:00" //K線時間
    }
}

注單成交通知

cmd: Deal

方向 : client <— server

  • 說明:
    • 這個Deal通知會由server端發送給client端, 讓client端知道注單已經成交並取得成交價格, 停利(贏)與停損(輸)價格
    • 此Deal通知會在成交後立刻通知client, 並且在1秒後再通知一次, 一張注單總共就通知2次
    • 此Deal通知可以應用在玩家送出注單後, client端可以透過收到此通知來得知注單已經成交, 並顯示注單的成交資料給玩家知道






















{
    "cmd": "Deal",
    "msgtime": "2023-05-29T16:50:02+08:00",
    "Data": {
        "symbol": "EURUSD", //商品代碼
        "orderID": "chq6eeh7e7eu7556bps0", //注單編號
        "ctime":"2023-05-29T08:50:01Z" //訂單建立時間
        "cts":1685350200 //訂單建立時間戳記(unix timestamp)
        "dtime": "2023-05-29T08:50:02Z", //成交時間
        "ts": 1685350202,  //成交時間戳記(unix timestamp)
        "dealprice": 1.07266, //成交價格
        "winprice": 1.07288,  //停利價格(贏)
        "loseprice": 1.07251,  //停損價格(輸)
        "winpercent": 0.02, //合約區間
        "expectwin": 100, //預期獲利
        "longshort": -1, //看漲看跌
        "auto":false,//設定自動續單, true:有續單, false:無續單
        "autotype":1,    //自動續單的類型 -1:輸續單 , 1:贏續單, 2:輸贏都續單
        "autodir":-1 //續單方向 1:同向, -1:反向
    }
}

注單結算進度

cmd: SettlePercent

方向 : client <— server

  • 說明:
    • 這個SettlePercent通知會由server端發送給client端, 讓client端知道注單目前距離結算的達成度
    • 這個訊息的主要重點在percent 這個欄位, 當注單距離停利價格越近, percent 會越接近 100(也就是100%達到停利價格), 當注單距離停損點價格越近, percent 會越接近 -100(也就是100%達到停損價格)
    • issettled 若為false, 代表該注單尚未達到結算的狀態, 若為true, 代表該注單已經達到結算狀態(已停損或已停利)










{
    "cmd": "SettlePercent",
    "msgtime": "2023-05-29T16:45:20+08:00",
    "Data": {
        "symbol": "EURUSD", //商品代碼
        "orderID": "chq69l97e7eu7556bpd0", //訂單編號
        "percent": -33,    //距離結算的百分比狀態
        "issettled": false //是否已結算
    }
}

最高最低價變化

cmd: DayHighLow

方向 : client <— server

  • 說明:
    • 這個高低變化的資料, 會在該商品的最高或最低價有變化的時候即時發出











{
    "cmd": "DayHighLow",
    "msgtime": "2023-07-17T16:07:26+08:00",
    "Data": {
        "symbol": "USDCHF", //商品代碼
        "high": 0.8627, //最新的最高價
        "low": 0.85922, //最新的最低價
        "ts": 1689581246, //資料產生的時間戳記
        "hltime": "2023-07-17T16:07:26+08:00" //資料產生的時間
    }
}
